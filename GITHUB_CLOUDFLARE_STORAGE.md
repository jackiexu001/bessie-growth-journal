# GitHub 和 Cloudflare 作为存储方案分析 📦

## 🤔 简单理解

### GitHub 是什么？
- **主要用途**：代码托管（存代码）
- **类比**：就像"代码版的网盘"
- **不适合**：存储大量照片/视频

### Cloudflare 是什么？
- **主要用途**：CDN 加速、网站保护
- **也有存储**：Cloudflare R2（对象存储）
- **类比**：就像"网站加速器 + 存储"

---

## 📦 GitHub 作为存储方案

### ❌ 不推荐使用 GitHub 存储照片/视频

#### 为什么不行？

1. **不是设计用来存文件的**
   - GitHub 是代码托管平台
   - 主要用来存代码，不是媒体文件

2. **有文件大小限制**
   - 单个文件最大 100MB
   - 超过 50MB 会警告
   - 超过 100MB 会被拒绝
   - **视频文件通常都超过 100MB**

3. **有仓库大小限制**
   - 推荐仓库 < 1GB
   - 超过 1GB 会影响性能
   - 超过 5GB 可能被限制

4. **不适合频繁上传**
   - GitHub 不是为频繁上传设计的
   - 每次上传需要提交代码
   - 不适合自动化上传

5. **违反服务条款**
   - GitHub 的服务条款不允许大量存储非代码文件
   - 可能被限制或封号

#### 结论
- ❌ **不适合存储照片/视频**
- ❌ **文件大小限制太严格**
- ❌ **可能违反服务条款**

---

## ☁️ Cloudflare R2 作为存储方案

### ✅ 可以！而且很好！

#### Cloudflare R2 是什么？

**Cloudflare R2 = Cloudflare 的对象存储服务**

**简单理解：**
- 就像"Cloudflare 版的网盘"
- 专门用来存储文件
- 和 AWS S3、腾讯云 COS 类似

---
