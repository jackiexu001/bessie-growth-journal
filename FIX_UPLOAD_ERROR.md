# 修复上传失败问题 🔧

## ⚠️ 问题说明

网站已部署成功，但上传图片和视频失败。

---

## 🔍 立即检查

### 步骤 1：查看 Netlify 函数日志（最重要）

#### 1.1 查看函数日志

1. **访问 Netlify**
   - https://app.netlify.com
   - 登录

2. **进入站点页面**
   - 点击站点 `bessie-growth-journal`

3. **查看函数日志**
   - 点击 **"Functions"**（函数）标签
   - 或者点击 **"Deploys"** → 选择最新部署 → **"Functions"**
   - 查看上传时的日志

#### 1.2 查看构建日志

1. **在站点页面**
2. **点击 "Deploys"** 标签
3. **点击最新的部署记录**
4. **查看 "Build log"**（构建日志）
5. **查找错误信息**

---

### 步骤 2：查看浏览器控制台

#### 2.1 打开开发者工具

1. **访问网站**：https://bessie.netlify.app
2. **按 F12 打开开发者工具**
3. **点击 "Console"**（控制台）标签

#### 2.2 尝试上传并查看错误

1. **尝试上传一个文件**
2. **查看控制台中的错误信息**
3. **记录错误信息**

#### 2.3 查看网络请求

1. **在开发者工具中**
2. **点击 "Network"**（网络）标签
3. **尝试上传文件**
4. **找到 `/api/upload` 请求**
5. **查看请求详情和响应**

---

### 步骤 3：检查环境变量

#### 3.1 在 Netlify 中检查

1. **在站点页面**
2. **点击 "Site configuration"** → **"Environment variables"**
3. **确认以下 4 个变量都存在：**
   - `STORAGE_TYPE`
   - `CLOUDINARY_CLOUD_NAME`
   - `CLOUDINARY_API_KEY`
   - `CLOUDINARY_API_SECRET`

#### 3.2 检查变量值

**确认：**
- `STORAGE_TYPE` = `cloudinary`（完全小写，没有空格）
- 其他三个变量的值正确（没有多余空格）

---

## 🔧 常见问题和解决方案

### 问题 1：环境变量未生效

**症状：**
- 上传失败
- 日志显示 "Cloudinary 配置不完整"

**解决：**
1. 检查环境变量是否正确配置
2. 确认选择了 "All scopes"
3. **重新部署站点**（重要！）

---

### 问题 2：Cloudinary API 密钥错误

**症状：**
- 上传失败
- 日志显示 Cloudinary API 错误

**解决：**
1. 检查 Cloudinary 控制台中的 API 密钥
2. 确认在 Netlify 中配置的密钥正确
3. 重新配置环境变量
4. **重新部署站点**

---

### 问题 3：Netlify 函数环境变量问题

**可能的问题：**
- 环境变量在构建时可用，但在运行时不可用
- Next.js 在 Netlify 上的环境变量处理方式不同

**解决：**
- 确保环境变量选择了所有作用域（All scopes）
- 重新部署站点

---

### 问题 4：文件大小限制

**症状：**
- 大文件上传失败

**解决：**
- 检查文件大小（应该 < 1024MB）
- 检查 Netlify 函数大小限制
