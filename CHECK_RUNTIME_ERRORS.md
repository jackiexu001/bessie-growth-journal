# 查看运行时错误 🔍

## ⚠️ 重要说明

你刚才看到的是 **构建日志**（Build Log），显示构建成功 ✅

但上传失败是 **运行时错误**，需要查看不同的日志！

---

## 🔍 查看运行时错误的步骤

### 步骤 1：查看浏览器控制台（最重要）

#### 1.1 打开开发者工具

1. **访问网站**：https://bessie.netlify.app
2. **按 F12** 打开开发者工具
3. **点击 "Console"**（控制台）标签

#### 1.2 尝试上传并查看错误

1. **尝试上传一个文件**
2. **查看控制台中的错误信息**
3. **记录所有红色错误信息**

**常见错误：**
- `Failed to fetch`
- `Network error`
- `500 Internal Server Error`
- `Cloudinary 配置不完整`

---

### 步骤 2：查看网络请求

#### 2.1 打开网络标签

1. **在开发者工具中**
2. **点击 "Network"**（网络）标签
3. **清空之前的请求**（点击 🚫 图标）

#### 2.2 尝试上传

1. **尝试上传一个文件**
2. **找到 `/api/upload` 请求**
3. **点击这个请求**

#### 2.3 查看请求详情

**查看以下信息：**

1. **Status（状态）**
   - 200 = 成功
   - 400 = 请求错误
   - 500 = 服务器错误

2. **Response（响应）**
   - 点击 "Response" 标签
   - 查看返回的错误信息

3. **Headers（请求头）**
   - 点击 "Headers" 标签
   - 查看请求详情

**截图或复制错误信息！**

---

### 步骤 3：查看 Netlify 函数日志（运行时）

#### 3.1 访问 Netlify

1. **访问**：https://app.netlify.com
2. **登录**
3. **点击站点** `bessie-growth-journal`

#### 3.2 查看函数日志

**方法 A：通过 Functions 标签**

1. **点击 "Functions"**（函数）标签
2. **查看最新的函数执行记录**
3. **点击执行记录查看日志**

**方法 B：通过 Deploys 标签**

1. **点击 "Deploys"**（部署）标签
2. **点击最新的部署记录**
3. **点击 "Functions"** 或 **"View function logs"**
4. **查看日志**

#### 3.3 查找错误

在日志中查找：
- `Cloudinary 配置不完整`
- `Cloudinary 上传错误`
- `Upload error`
- 任何红色错误信息

**注意：这是运行时日志，不是构建日志！**
