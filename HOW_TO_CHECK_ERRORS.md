# 如何查看上传错误 🔍

## ⚠️ 重要说明

你刚才看到的是 **构建日志**（Build Log），显示构建成功 ✅

但上传失败是 **运行时错误**，需要查看不同的日志！

---

## 🔍 查看错误的 3 个步骤

### 步骤 1：查看浏览器控制台（最重要！）

#### 1.1 打开开发者工具

1. **访问网站**：https://bessie.netlify.app
2. **按 F12** 打开开发者工具
3. **点击 "Console"**（控制台）标签

#### 1.2 尝试上传并查看错误

1. **尝试上传一个文件**
2. **查看控制台中的错误信息**
3. **记录所有红色错误信息**

**你会看到类似这样的错误：**
- `Upload error: ...`
- `上传失败详情: ...`
- `Cloudinary 配置不完整`
- `Failed to fetch`

**截图或复制这些错误信息！**

---

### 步骤 2：查看网络请求（非常重要！）

#### 2.1 打开网络标签

1. **在开发者工具中**
2. **点击 "Network"**（网络）标签
3. **清空之前的请求**（点击 🚫 图标）

#### 2.2 尝试上传

1. **尝试上传一个文件**
2. **找到 `/api/upload` 请求**
3. **点击这个请求**

#### 2.3 查看请求详情

**查看以下信息：**

1. **Status（状态码）**
   - 200 = 成功 ✅
   - 400 = 请求错误 ❌
   - 500 = 服务器错误 ❌

2. **Response（响应内容）**
   - 点击 "Response" 标签
   - 查看返回的错误信息
   - **这是最重要的！**

**你会看到类似这样的响应：**
```json
{
  "error": "上传失败，请重试",
  "details": "Cloudinary 配置不完整",
  "storageType": "cloudinary",
  "hasCloudinaryConfig": false
}
```

**截图或复制这个响应！**

---

### 步骤 3：查看 Netlify 函数日志

#### 3.1 访问 Netlify

1. **访问**：https://app.netlify.com
2. **登录**
3. **点击站点** `bessie-growth-journal`

#### 3.2 查看函数日志

**方法 A：通过 Functions 标签**

1. **点击 "Functions"**（函数）标签
2. **查看最新的函数执行记录**
3. **点击执行记录查看日志**

**方法 B：通过 Deploys 标签**

1. **点击 "Deploys"**（部署）标签
2. **点击最新的部署记录**
3. **点击 "Functions"** 或 **"View function logs"**
4. **查看日志**

#### 3.3 查找错误

在日志中查找：
- `Cloudinary 配置不完整`
- `Cloudinary 上传错误`
- `Upload error`
- `使用存储类型: ...`
- `配置检查: ...`

**注意：这是运行时日志，不是构建日志！**

## 🎯 常见错误和解决方案

### 错误 1：Cloudinary 配置不完整

**症状：**
- 响应显示 `hasCloudinaryConfig: false`
- 日志显示 "Cloudinary 配置不完整"

**解决：**
1. 检查 Netlify 环境变量
2. 确认所有变量都已配置
3. 确认选择了 "All scopes"
4. 重新部署站点

---

### 错误 2：Cloudinary API 密钥错误

**症状：**
- 响应显示 Cloudinary API 错误
- 日志显示 "Invalid API key"

**解决：**
1. 检查 Cloudinary 控制台中的 API 密钥
2. 确认在 Netlify 中配置的密钥正确
3. 重新配置环境变量
4. 重新部署站点

---

### 错误 3：网络错误

**症状：**
- 控制台显示 "Failed to fetch"
- 网络请求显示 "Network error"

**解决：**
1. 检查网络连接
2. 检查 Netlify 函数是否正常运行
3. 查看 Netlify 函数日志
