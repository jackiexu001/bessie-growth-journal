# 故障排除指南

## 视频上传失败问题

### 问题描述
上传视频时出现失败错误。

### 可能的原因

1. **文件大小超过限制**
   - 当前代码限制为 1024MB（1GB）
   - Next.js 默认的请求体大小限制约为 4.5MB，可能在实际使用中先于代码限制生效

2. **服务器内存不足**
   - 大文件上传需要足够的内存

3. **文件格式不支持**
   - 确保视频格式是常见的格式（MP4, MOV, AVI 等）

### 解决方案

#### 方案 1: 检查文件大小
- 确保文件小于 1024MB（1GB，当前代码限制）
- 如果文件太大，可以使用视频压缩工具压缩后再上传

#### 方案 2: 使用流式上传（推荐）
对于大文件，建议使用流式上传而不是一次性加载到内存。

#### 方案 3: 检查服务器日志
查看终端中的服务器日志，查找具体的错误信息：
- 文件大小
- 错误类型
- 堆栈跟踪

### 调试步骤

1. **检查浏览器控制台**
   - 打开浏览器开发者工具（F12）
   - 查看 Console 标签页的错误信息
   - 查看 Network 标签页，找到上传请求，查看响应详情

2. **检查服务器日志**
   - 查看运行 `npm run dev` 的终端窗口
   - 查找 "Upload error:" 开头的错误信息

3. **测试小文件**
   - 先尝试上传一个小的测试视频（< 5MB）
   - 如果小文件可以上传，说明问题是大文件限制

### 临时解决方案

如果急需上传大视频文件，可以：

1. **压缩视频**
   - 使用 HandBrake、FFmpeg 等工具压缩视频
   - 降低分辨率或比特率

2. **分块上传**
   - 将大视频分割成多个小段
   - 分别上传后手动合并（需要额外开发）

3. **使用云存储**
   - 集成 Cloudinary、AWS S3 等云存储服务
   - 这些服务专门处理大文件上传

### 长期解决方案

建议集成专业的文件上传服务：
- **Cloudinary**: 支持视频上传和自动优化
- **AWS S3**: 对象存储服务
- **Vercel Blob**: Vercel 的存储服务

这些服务可以：
- 处理大文件上传
- 自动生成视频缩略图
- 提供 CDN 加速
- 自动压缩和优化
